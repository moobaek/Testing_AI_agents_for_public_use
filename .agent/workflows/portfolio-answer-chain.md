# 포트폴리오 분석 체인 워크플로우

포트폴리오에 대한 심층 분석이 필요할 때 이 워크플로우를 따릅니다.

## Step 1: 포트폴리오 구조 분석

**목적**: 포트폴리오 전체 구조 파악

**분석 항목**:
- 폴더 구조 및 파일 구성
- 문서 타입 식별 (프로젝트, 가이드, 템플릿 등)
- ID 시스템 매핑

**핵심 경로**:
```
portfolio/portfolio_docs/
├── 00_*.md                    # 핵심 가이드 문서
├── 0X_*.md                    # 주요 개요 문서
├── Phase_*_*/                 # Phase별 가이드
├── prompts/                   # AI 프롬프트 시스템
├── templates/                 # 문서 템플릿
└── assets/                    # 자산 파일
```

## Step 2: 문서 내용 분석

**목적**: 질문 관련 문서 내용 심층 분석

**분석 항목**:
- 섹션 구조 분석
- 핵심 키워드 추출
- 관련 내용 식별

**주요 문서**:
| 문서 | 내용 |
|------|------|
| `00_Personal_Profile.md` | 개인 프로필, 핵심 철학 |
| `02_Projects_Overview.md` | 20개+ 프로젝트 개요 |
| `Architecture_Overview.md` | 아키텍처 상세 (AMS, DPS, FMEA 등) |
| `04_Academic_Publications.md` | 논문 9편 목록 |
| `권순룡_포트폴리오_통합문서.md` | 전체 통합 요약 |

## Step 3: 관계 매핑

**목적**: 문서 간 관계 및 프로젝트 연결 파악

**관계 유형**:
- 프로젝트 → 기술 스택
- 프로젝트 → 성과 (GS 인증, 논문, 특허)
- 프로젝트 → 납품처
- 문서 → 문서 참조

**핵심 관계**:
```
AMS → Neo4j, Python 49개 모듈
AMS → GS 1등급, 논문 3편, 특허
AMS → 세아특수강, 포미아 납품
DPS → 5층 아키텍처, Neo4j
FMEA → 8개 Sub-Agent, Multi-Agent
```

## 분석 결과 활용

분석 완료 후:
1. 관계 그래프 기반 답변 생성
2. 관련 문서 링크 제공
3. 순룡 페르소나 스타일로 답변
