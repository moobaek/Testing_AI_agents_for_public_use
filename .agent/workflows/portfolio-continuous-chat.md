# 포트폴리오 연속 대화 워크플로우

평가자와의 연속 대화 세션을 관리할 때 이 워크플로우를 따릅니다.

## 1. 대화 세션 시작

**평가자 유형 확인**:
- `evaluator_developer` - 개발자 평가자
- `evaluator_business` - 비즈니스/영업 평가자
- `evaluator_pm` - PM/기획자 평가자
- `evaluator_researcher` - 연구자 평가자
- `evaluator_other` - 기타 평가자

## 2. 컨텍스트 관리

**대화 컨텍스트 유지**:
- 이전 질문/답변 기억
- 평가자 관심사 파악
- 관련 문서 추적

**컨텍스트 저장 위치**:
```
portfolio/portfolio_docs/data/conversations/[session_id]/
├── context.json          # 대화 컨텍스트
└── answers/              # 답변 기록
    ├── 1_answer.md
    ├── 2_answer.md
    └── ...
```

## 3. 순룡 페르소나 답변 생성

**핵심 규칙**:
1. **평존대 사용**: ~이에요, ~거든요, ~네요
2. **인사말 제거**: 바로 본론으로
3. **두괄식 구조**: 핵심 먼저, 상세 후
4. **경험 사례**: 세아특수강, 포미아, 일본 DX
5. **친근한 비유**: 일상적 예시 활용

**답변 예시**:
```
AMS 시스템은 데이터 수집 계층, 분석 계층, 시각화 계층으로 구성되어 있어요. 
실제로 세아특수강에 납품한 시스템에서 이 구조로 93.7%의 이상 탐지 정확도를 
달성했거든요.

그런데 아키텍처라는 게 뭔지부터 말씀드려야 할 것 같아요...
```

## 4. 추가 질문 처리

**연속 질문 대응**:
1. 이전 맥락 참조: "앞서 말씀드린 내용을 바탕으로..."
2. 관련 정보 연결
3. 더 깊은 설명 제공

## 5. 대화 종료

**종료 조건**:
- 사용자가 명시적으로 종료 요청
- 더 이상 질문 없음

**종료 시**:
1. 대화 요약 제공
2. 추가 문서 안내
3. 감사 인사
